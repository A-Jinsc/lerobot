⚡ 优化点详解

本次优化在基线代码基础上做的三处改进

Opt 1

SDPA 替代手写 Eager Attention

PyTorch 的 scaled_dot_product_attention 自动选择最优后端（MPS/Accelerate）， 减少中间 tensor 创建和内存拷贝，比手写 expand+reshape+matmul+softmax 更高效。

# Baseline: 手写实现
att = torch.matmul(q, k.transpose(2,3))
probs = F.softmax(att, dim=-1)
out = torch.matmul(probs, v.permute(...))
↓
# Optimized: SDPA
out = F.scaled_dot_product_attention(q, k, v)
Opt 2

StaticCache 预分配 KV 缓存

基线在每次 denoise step 中通过 torch.cat([past_kv, new_kv]) 拼接 KV， O(n) 拷贝开销。优化版预先分配固定大小 tensor，直接写入指定位置， 避免每次 cat 的内存分配和拷贝。

# Baseline: 每次动态拼接
k = torch.cat([past_kv["k"], new_k], dim=1)
v = torch.cat([past_kv["v"], new_v], dim=1)
↓
# Optimized: 预分配写入
_static_k[0,layer, :,offset:offset+len] = k
Opt 3

分块 Cross-Attention

在 denoise loop 中，expert 的 cross-attention 查询可以分块处理 （如每块 16 个 action token），减少单次计算的峰值内存占用， 对 CPU 上的 memory bandwidth 友好。

# 分块处理 action tokens
chunk_size = 16
for i in range(0, action_len, chunk_size):
  chunk_q = q[:, i:i+chunk_size]
  out_chunk = sdpa(chunk_q, cached_k, cached_v)
  outputs.append(out_chunk)
